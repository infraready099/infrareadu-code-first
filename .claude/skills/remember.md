# Skill: Remember

Quick mid-session memory capture. Use this when something important comes up that should persist across conversations — without doing the full /reflect flow.

## What to capture

The user will provide something to remember. It will be one of:

1. **A fact or decision** → save to `memory/decisions-log.md`
2. **A mistake or rule** → save to `memory/lessons-learned.md`
3. **Project state or priority** → save to `memory/MEMORY.md`
4. **User preference** → save to `memory/MEMORY.md` under a Preferences section

## Steps

1. Identify which memory file(s) the item belongs in.
2. Read the relevant file first (required before editing).
3. Append the new entry in the correct format — do NOT duplicate existing entries.
4. Confirm to the user: "Saved to memory/[file].md"

## Formats

**decisions-log.md entry:**
```markdown
### YYYY-MM-DD | <Title>
**Decision:** <What>
**Why:** <Rationale>
**Trade-offs:** <What we give up>
**Outcome:** Active
```

**lessons-learned.md entry:**
```markdown
### YYYY-MM-DD | <Title>
**What went wrong:** <Symptom>
**Root cause:** <Why>
**Fix:** <Resolution>
**Rule:** <One-line rule>
```

**MEMORY.md update:**
Edit the relevant section in place (Current Sprint, Open Questions, Next Priorities, etc.)
